# 日期星期与未来多年周末/节假日概率排行（纯前端）

无需 npm、无需构建，`index.html + styles.css + app.js` 直接静态运行。

## 本地运行

```bash
python -m http.server 8000
```

访问：`http://localhost:8000`

## GitHub Pages 部署（Deploy from branch）

1. 推送代码到 GitHub（如 `main`）。
2. 仓库 `Settings → Pages`。
3. `Build and deployment` 选择 `Deploy from a branch`。
4. Branch 选择 `main`，Folder 选择 `/root`。
5. 保存后访问生成的 Pages 链接。

## 功能

### A. 单个日期判定
- 输入“年 / 月 / 日”三格（不需要连字符，月日可填 `3` 或 `03`），输出：
  - 周几（中英）
  - 是否周末
  - 是否规则节假日（中国法定节日本体日口径）
  - 是否日历模式休息日（若已加载 off/work）
- 星期算法使用 Sakamoto 纯整数算法，避免 `new Date("YYYY-MM-DD")` 字符串解析时区偏移。

### B. 月日未来 N 年星期
- 输入“月 / 日”两格 + `N(1~50)` + `startYear`（月日支持 `3` 或 `03`）。
- 输出 Year/Date/Weekday 表格。
- `02-29` 在非闰年显示 `N/A`。

### C. 月日区间概率排行
- 输入区间起点与终点的“月 / 日”两格，支持跨年区间（例如 `12/20 ~ 1/10`）。
- 输出每个 `MM-DD` 在未来 N 年的命中概率排行（带明细、过滤、TopK、复制 TSV）。
- 维度由“节假日模式”决定：
  - 规则模式：周末概率 / 节假日概率 / 休息日概率（周末 OR 节假日）
  - 日历模式：off 概率 / 休息日概率（off + 周末 - work）/ 法定节日本体日概率（规则对照）

### D. 节假日放假概率（中国口径）

#### 1) 规则模式（默认，可预测）
按法定节日本体日判定，不模拟调休拼假：
- 元旦：`01-01`
- 春节：农历除夕、正月初一、初二、初三
- 清明：清明当日（内置清明日期公式）
- 劳动节：`05-01`、`05-02`
- 端午：农历五月初五
- 中秋：农历八月十五
- 国庆：`10-01`、`10-02`、`10-03`

#### 2) 日历模式（最真实）
加载 JSON（粘贴或上传），格式：

```json
{
  "2026": {
    "off": ["2026-01-01", "2026-10-02"],
    "work": ["2026-10-10"]
  }
}
```

判定规则：
- 在 `work`：强制工作日（即使是周末）
- 否则在 `off`：强制休息日
- 否则按周末规则

### E. 日期计算（纪念日 / N 天后）
- 输入起始日期（年/月/日）和相隔天数（可正可负，整数）。
- 立即计算目标日期，并输出目标日期是周几。
- 计算采用 `Date.UTC(y, m-1, d)` + UTC 组件，避免时区偏移和字符串解析误差。

## 2026 示例 JSON（可在页面一键填充）

页面内置了一个 2026 示例数据（`off/work`）用于演示“解析与判定流程”。
该示例标注为“来源口径：国务院办公厅《2026年部分节假日安排的通知》”，实际使用时请以当年官方最新发布版本自行核对/替换。

## 自检

页面“运行自检”按钮会验证：
1. 10 条已知日期 weekday 结果。
2. `2024-02-10` 能识别为春节（农历正月初一）。
3. 示例 2026 JSON 在 off/work 判定逻辑中生效。

## 农历实现说明

- 本项目内置了纯 JS 农历换算实现（1900-2100），使用常见 `lunarInfo` 数据表算法思路，直接放在 `app.js`，无外部 CDN / npm 依赖。
- 清明日期使用 1900-2099 常见近似公式并包含常见修正年。
